# Aula 01 - Conceitos de Arquitetura de Software
    Uma boa arquitetura permite a evolução do sistema de forma sustentável e facilita manutenção e extensibilidade a longo prazo e uma arquitetura bem projetada contribui para a qualidade do software garantindo requisitos não funcionais como desempenho segurança e escalabilidade.
    
    A arquitetura de um software pode ser dividida em três elementos:
        1. Os componentes que são os blocos de construção do sistema que podem ser módulos classes serviços também temos as conexões que são as interações entre os componentes como comunicação dependências.
        
        2. Fluxo de dados 
        
        3. As decisões de design que são as escolhas tomadas durante a definição da arquitetura como padrões tecnologias e abordagens utilizadas/
        
    Os primeiros princípios fundamentais da arquitetura de software que devemos seguir são os princípios Sid ou Solid que são o conjunto de cinco princípios que guiam o design de software orientado a objetos são eles o 
    
        1. Princípio da responsabilidade única 
        
        2. O princípio do aberto ou fechado 
        
        3. O princípio da substituição de liskov 
        
        4. O princípio da segregação de interfaces 
        
        5. O princípio da inversão de dependência 
    
    SOLID: 
        1. O "S" é usando do princípio da responsabilidade única srp acreditamos que uma classe deve ter apenas uma razão para mudar e assim a tratamos
        
        2. O "O" é o princípio do aberto ou fechado, uma classe deve estar aberta para extensão mas fechada Para modificação 
        
        3. O "L" é o princípio da substituição de liskov, tratamos objetos de um tipo base substituindo-os por objetos de um tipo derivado sem afetar a integridade do programa 
        
        4. O "I" é o princípio da segregação de interfaces, acreditamos que muitas interfaces especializadas são melhores do que uma interface única
        
        5. O "D" é de princípio da inversão de dependência, os módulos de alto nível não devem depender de módulos de baixo nível ambos devem depender de Abstrações 
        
    Outro princípio fundamental da arquitetura de software é o princípio do encapsulamento e coesão, enquanto o encapsulamento enfatiza a ocultação de detalhes internos dos componentes e a exposição de uma interface clara e consistente para interação, a coesão é a medida de quão fortemente os elementos de um componente estão relacionados entre si, os componentes com alta coesão por exemplo são mais fáceis de entender manter e reutilizar 
    
    Mais um princípio fundamental que devemos utilizar é o da modularidade e reutilização de código, a modularidade envolve dividir um sistema em módulos Independentes, que podem ser ouvidos testados mantidos separadamente, a reutilização de código é a prática de usar componentes existentes em diferentes partes dos sistemas economizando tempo e esforço.


# Aula 02 - Padrões de Arquitetura de Software

    Padrões de arquitetura de desenvolvimento de software, são soluções comprovadas e recorrentes para problemas comuns enfrentados no projeto e na construção de aplicativos e sistemas de software. Esses padrões fornecem diretrizes e melhores práticas para organizarmos o código, definir a estrutura do aplicativo, gerenciar a interação entre componentes e lidar com a escalabilidade de manutenção e extensibilidade.
    
    Os padrões de arquitetura de software visam alcançar objetivos como separação de responsabilidades, reutilização de código, baixo acoplamento, alta Coesão, escalabilidade, manutenibilidade e extensibilidade do sistema, eles ajudam a melhorar a qualidade do software facilitando a colaboração entre os membros da equipe e permitem que o desenvolvimento seja mais eficiente. 
    
    Existem diversos padrões de arquitetura de software amplamente conhecidos como 
        1. MVC - Model View Controller 
        
        2. MVVM - Model View View Model 
        
        3. MVP - Model View Presenter 
        
        4. Arquitetura em camadas 
        
        5. Arquitetura cliente servidor 
        
        6. Arquitetura de microsserviços 
        
        7. Arquitetura orientada a serviços 
        
        8. Arquitetura de pipeline 
        
        9. Arquitetura em eventos
        
        10. Arquitetura hexagonal 
        
    Esses padrões comumente utilizados, fornecem uma estrutura para projetar e organizar o código definindo como as diferentes partes do sistema interagem entre si.

# Aula 03 - Detalhamento de Padrões de Arquitetura de Software Moderno

    Existem diversos padrões arquiteturais que podem ser aplicados no desenvolvimento de sistemas. Esses padrões tem como objetivo organizar e estruturar o código, de forma a facilitar a manutenção escalabilidade e reutilização.
    
    Principais padrões de arquitetura utilizados no desenvolvimento web e mobile: 
    
        1. MVC - Ele divide o sistema ou aplicativo em três componentes principais, o Model ou modelo, a view ou visão, e o Controller ou controle. O Model representa os dados e as regras de negócio, a view é responsável pela apresentação dos dados aos usuários Ou seja a interface gráfica e o Controller trata das interações do usuário e coordena a comunicação entre o modelo e a visão. 
        
        Ele é amplamente utilizado em várias plataformas e linguagens de programação, o padrão mvc no desenvolvimento web e mobile segue a mesma ideia fundamental, mas existem alguns detalhes em sua implementação específica para cada plataforma. 
        
        No desenvolvimento web o model contém as regras de negócios e gerencia os dados do aplicativo, ele representa as regras e operações relacionadas aos dados, como validação, manipulação, consulta e persistência. No desenvolvimento web o Model geralmente inclui a camada de acesso a banco de dados ou serviços de api, a view captura essa interação e envia uma requisição para o controlador, o controlador processa a requisição atualiza o Model e realiza qualquer lógica ou regra necessária e seleciona a View apropriada para renderizar a resposta, a View recebe os dados atualizados do Model e os apresenta ao usuário.
        
        No desenvolvimento mobile o padrão mvc é uma estrutura similar ao desenvolvimento web mas com algumas adaptações para a plataforma mobile, o Model é responsável pela lógica de negócios manipulação de dados e gerenciamento do estado do aplicativo, ele representa as regras de negócio e os dados subjacentes, no desenvolvimento mobile o modelo geralmente inclui acesso a dados chamados de api e manipulação de armazenamento local, a view no contexto mobile é responsável pela interface do usuário e pela exibição dos dados do módelo, ela é a camada que renderiza os elementos da interface como os botões, Campos de texto, listas e outros componentes visuais. 
        No desenvolvimento mobile a view é implementada usando elementos de interface nativos ou componentes específicos do Framework utilizado, o Controller é responsável por receber as interações do usuário na view e processá-las e atualizar o Model, conforme necessário ele coordena as ações do usuário e lida com a lógica de controle. No desenvolvimento mobile o controlador é implementado como uma classe ou componente que gerencia as interações da View e atualiza o módel.
